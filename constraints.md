# Lifeline Church Website — Constraints & Guidelines

## Technology Constraints

- **No Node.js, npm, or any server-side runtime.** This is a purely static website — plain HTML, CSS, and vanilla JavaScript only. No build steps, no bundlers, no frameworks.
- **No React, Vue, Angular, Next.js, Vite, or similar.** Keep it simple so anyone can edit and deploy it.
- **No Tailwind CSS or CSS preprocessors (Sass, Less).** Use plain CSS with CSS custom properties (variables) for theming.
- **No server required to run.** The site should work by opening `index.html` directly in a browser, or by hosting on any basic static file host (GitHub Pages, Netlify, shared hosting, etc.).

## Content Management

- **All editable content lives in `siteConfig.json`.** Non-technical users should be able to update the website by editing this single JSON file — no coding knowledge needed.
- **Photo gallery categories and file lists live in `gallery.json`.** This file is auto-generated by `scan_photos.js` and lists every photo category folder and its files.
- **The `liveVidEmbed.txt` file is legacy.** The YouTube video ID is now stored in `siteConfig.json` under `liveVideoEmbed.youtubeVideoId`. The old file is kept as a fallback only.

## What `siteConfig.json` Controls

| Section | Config Key | What It Changes |
|---|---|---|
| Colors/theme | `colors` | Every color across the site |
| Church info | `churchInfo` | Name, tagline, service times, phone, email, address |
| Welcome/Map section | `welcomeSection` | Heading and schedule text |
| About Us | `aboutUs` | Heading, paragraph, images |
| Photos header | `photosSection` | Section heading and description |
| Upcoming Events | `events.list` | Add/remove any number of events freely |
| Ministries | `ministries` | Heading, text, YouTube/Facebook links |
| Video embed | `liveVideoEmbed` | YouTube video ID |
| Give/Donate | `give` | Heading, description, giving options |
| Social links | `socialLinks` | Facebook, Instagram, YouTube URLs |
| Map embed | `mapEmbed` | Google Maps embed URL |
| Hero slideshow | `landingSlideshow` | Background images and timing |

## File Structure Rules

- **`photos/`** — Contains subfolders (e.g., `cat1/`, `cat2/`) for photo categories. Drop images into these folders and run `scan_photos.js` to update `gallery.json`.
- **`land_imgs/`** — Landing/hero background slideshow images. Referenced by path in `siteConfig.json`.
- **`abt_us_imgs/`** — About Us section images. Referenced by path in `siteConfig.json`.
- **`icons/`** — Logo and favicon assets.

## Design Constraints

- Maintain a warm, clean, modern aesthetic with the existing design system.
- Use Google Fonts (Poppins for headings, Lato for body) — loaded via CDN, no local font files.
- Use Font Awesome 5 icons via CDN.
- All photos must preserve their original aspect ratio (`object-fit: cover` for thumbnails, `object-fit: contain` for lightbox).
- The site must be fully responsive (desktop, tablet, mobile).

## Deployment

- Can be hosted on any static file server — no server-side processing needed.
- No build step required. Just upload the files as-is.
