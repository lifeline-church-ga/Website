# Lifeline Church Website — Constraints & Guidelines

## Technology Constraints

- **No Node.js, npm, or any server-side runtime.** This is a purely static website — plain HTML, CSS, and vanilla JavaScript only. No build steps, no bundlers, no frameworks.
- **Exception:** `repoSetup.js` is a Node.js utility script run locally to regenerate `gallery.json`. It is never used at runtime.
- **No React, Vue, Angular, Next.js, Vite, or similar.** Keep it simple so anyone can edit and deploy it.
- **No Tailwind CSS or CSS preprocessors (Sass, Less).** Use plain CSS with CSS custom properties (variables) for theming.
- **No server required to run.** The site should work by opening `index.html` directly in a browser, or by hosting on any basic static file host (GitHub Pages, Netlify, shared hosting, etc.).

## Content Management

- **All editable content lives in the `content/` folder as plain text files.** Non-technical users update the website by editing these simple `.txt` files — no coding or JSON syntax needed.
- **Photo gallery categories and file lists live in `gallery.json`.** This file also contains `land_imgs` and `abt_us_imgs` arrays. Auto-regenerated by running `node repoSetup.js`.
- **The YouTube video auto-updates every Sunday at 3 PM EDT.** A GitHub Actions workflow (`.github/workflows/update-video.yml`) fetches the latest video and updates `content/video.txt` automatically. Requires a `YOUTUBE_API_KEY` GitHub Secret.

## Content Files (`content/` folder)

| File | What It Controls |
|---|---|
| `church_info.txt` | Church name, tagline, service times, contact info, welcome section |
| `about_us.txt` | About Us heading and paragraph |
| `events.txt` | Section heading + event list (blank lines separate events) |
| `ministries.txt` | Ministries heading, text, and links |
| `video.txt` | YouTube video ID (auto-updated by GitHub Actions) |
| `give.txt` | Give section heading, description, and options |
| `social_links.txt` | Facebook, Instagram, YouTube URLs |
| `map.txt` | Google Maps embed URL |
| `colors.txt` | Site color theme |
| `settings.txt` | Hero display, slideshow timing, scroll snap, photos header |

### Text File Format

All content files use a simple `key: value` format:
```
name: Lifeline Church
tagline: Reaching the unreached
```

For lists (events, give options), items are separated by **blank lines**:
```
heading: Upcoming Events

title: Sunday Service
description: Join us every Sunday at 5 PM.

title: Bible Study
description: Wednesday evenings at 7 PM.
```

## File Structure Rules

- **`photos/`** — Contains subfolders for photo gallery categories. Drop images into a subfolder and run `node repoSetup.js`.
- **`land_imgs/`** — Hero/landing background slideshow images. Scanned by `repoSetup.js` into `gallery.json`.
- **`abt_us_imgs/`** — About Us section images. Scanned by `repoSetup.js` into `gallery.json`.
- **`icons/`** — Logo and favicon assets.
- **`content/`** — Editable text files for all website content.

## Adding New Photos

1. Copy the photo into the appropriate folder (e.g., `photos/outreach/`).
2. Run `node repoSetup.js` from the project root to regenerate `gallery.json`.
3. For new hero/slideshow images, add them to `land_imgs/` and re-run `repoSetup.js`.
4. For new About Us images, add them to `abt_us_imgs/` and re-run `repoSetup.js`.

## Design Constraints

- Maintain a warm, clean, modern aesthetic with the existing design system.
- Use Google Fonts (Poppins for headings, Lato for body) — loaded via CDN.
- Use Font Awesome 5 icons via CDN.
- All photos must preserve their original aspect ratio (`object-fit: cover` for thumbnails, `object-fit: contain` for lightbox).
- The site must be fully responsive (desktop, tablet, mobile).

## Deployment

- Hosted on **GitHub Pages**, Netlify, or any static file server — no server-side processing needed.
- No build step required. Just upload the files as-is.
